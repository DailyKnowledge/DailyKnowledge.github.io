A local optimum can hide a better solution nearby

A choice can be the best available from its current neighborhood while still being worse than another option farther away.

Takeaway

When small improvements stop helping, ask whether I am trapped on the wrong hill.

What I learned

A local optimum is the best point within a nearby set of options. It can feel like the best solution because every small move makes things worse, even though a better solution exists somewhere else.

Why it matters

This explains why steady improvement can still get stuck. If I only make small adjustments from the current position, I may never cross the valley needed to reach a better peak.

A simple example

A team may keep polishing an old workflow because each small tweak helps a little. A completely different workflow might be better, but getting there would require a temporary disruption.

How I can use it

  • Notice when incremental changes have stopped producing meaningful gains.
  • Try larger experiments when small moves all look worse.
  • Compare the current path with alternatives, not only nearby tweaks.
  • Accept short-term friction when it opens a better long-term option.
